# Agent Instructions

1. All documentation is in the `docs/` folder.
2. Do not delete any database files
3. Ensure all git commands are reversible.
4. Always plan first before executing.
5. Always update `docs/changelog.md` after any changes. Use the following format:
   - **Date > File name > methods or functions > what the change does**
   - Each change should be on a new bullet point
6. Before installing dependencies or creating additional files, get user permission and explain why they are needed.
6. After each feature push to github git add ., git commit -m "feature name", git push
